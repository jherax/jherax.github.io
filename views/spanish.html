Namespacing

En muchos lenguajes de programación, el "namespacing" o espacio de nombres, es una técnica empleada para evitar colisiones con otros objetos o variables en el ámbito global. Además de eso, esta técnica es de gran ayuda cuando necesitamos organizar bloques de funcionalidad en grupos fácilmente manejables e identificables.

Las variables globales deberían estar reservadas únicamente para objetos con una relevancia a nivel de sistema y deberían estar dentro de un espacio de nombres para evitar ambigüedades y minimizar el riesgo de colisión de nombres. En pocas palabras, esto significa que debería evitar en lo posible crear objetos globales a menos que sea absolutamente necesario.

Es crítico e importante salvaguardar nuestro código de fallas en el caso de que otro script en la página esté usando los mismos nombres de variables o funciones que hemos definido. Para solucionar éste y otros problemas, sugiero que hagamos uso del Patrón Module utilizando la inyección del espacio de nombres (namespacing). De este modo, nuestro código es protegido del ámbito global mediante una función que encapsula la lógica (usualmente un IIFE) y expone un objeto que representa la interface pública del módulo.


//Esta función encapsulada se llama IIFE
//Immediately-Invoked Function Expressions
var MODULE = (function () {
  var api = {},
      privateVar;

  function privateFunction() {}
  function exposedFunction() {}

  //expone la interface pública
  api.publicMethod = exposedFunction;
  return api;
}());

Este namespace define configuraciones globales que son usadas por algunos métodos.

nsEvents (eventname, namespace)

Construye el nombre del evento, añadiendo un punto y el @namespace al final de @eventname.

Enlazar y desenlazar eventos es un patrón común en la creación de plugins en jQuery, de manera que se pueda manejar las acciones ejecutadas por un evento, ¿pero que pasa si tengo más de un event-handler enlazado al mismo evento y quiero hacer algo específicamente con sólo uno de ellos?

Event namespacing es una técnica que provee una forma de administrar event-handlers específicos.
Recomiendo que lean los artículos: Namespace your events y jQuery event names and namespaces

Retorna String

  • eventname: String Nombre del evento. Puede ser una lista separada por espacios.
  • namespace: String Namespace del evento / lista de eventos.

fnAddScript (options)

Agrega dinámicamente un script. Éste método es útil para insertar código JavaScript desde un archivo externo.

Nota: Por defecto el recurso es cargado asíncronamente a través de jQuery.ajax, en cuyo caso, el método retorna un objeto jqXHR que implementa la interface Promise proporcionando los métodos del objeto Deferred. Tenga en cuenta que al utilizar jQuery.ajax no podrá solicitar recursos alojados en otro dominio, o utilizar el protocolo file:///, a menos que configure el método .ajax() para tal caso (vea jQuery.ajaxPrefilter)

Si configura createTag: true entonces se crea el elemento <script>, en cuyo caso el método retorna el DOM Element del <script> creado. El <script> es insertado antes del último script ejecutado ó antes del script especificado por @options.before

Retorna Element o jqXHR

  • options: String, Object: especifica el archivo a cargar.
    Si es String, entonces contiene la ruta del archivo JavaScript a cargar.
    Si es Object, entonces permite configurar las siguientes propiedades:
    • src: String default: null Especifica la URL del archivo a cargar.
    • async: Boolean default: true Determina si el script será ejecutado asíncronamente.
    • createTag: Boolean default: false Determina si se debe crear el elemento <script>
    • charset: String default: null Define la codificación de caracteres que utiliza el script.
    • onload: Function default: null Callback que se ejecuta después que el script ha sido cargado y ejecutado.
    • before: String default: null Indica en donde insertar el elemento <script>. Puede especificar parte de la ruta del archivo en el atributo src de alguno de los <script> en el documento.
