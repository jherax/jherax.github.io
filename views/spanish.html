Namespacing

En muchos lenguajes de programación, el "namespacing" o espacio de nombres, es una técnica empleada para evitar colisiones con otros objetos o variables en el ámbito global. Además de eso, esta técnica es de gran ayuda cuando necesitamos organizar bloques de funcionalidad en grupos fácilmente manejables e identificables.

Las variables globales deberían estar reservadas únicamente para objetos con una relevancia a nivel de sistema y deberían estar dentro de un espacio de nombres para evitar ambigüedades y minimizar el riesgo de colisión de nombres. En pocas palabras, esto significa que debería evitar en lo posible crear objetos globales a menos que sea absolutamente necesario.

Es crítico e importante salvaguardar nuestro código de fallas en el caso de que otro script en la página esté usando los mismos nombres de variables o funciones que hemos definido. Para solucionar éste y otros problemas, sugiero que hagamos uso del Patrón Module utilizando la inyección del espacio de nombres (namespacing). De este modo, nuestro código es protegido del ámbito global mediante una función que encapsula la lógica (usualmente un IIFE) y expone un objeto que representa la interface pública del módulo.

  //Esta función encapsulada se llama IIFE
  //Immediately-Invoked Function Expressions
  var MODULE = (function() {
    var api = {};
    var privateVar;
    function privateMethod1() {}
    function privateMethod2() {}
    //expone la interface pública
    api.publicMethod = privateMethod2;
    return api;
  }());

jsu.settings

Este namespace define configuraciones globales que son usadas por algunos métodos.

handlerExist (dom, eventname, namespace)

Determina si un manejador de eventos fue creado previamente especificando un namespace.
Note: Event namespacing es una técnica que nos permite manejar tareas de una forma diferente conforme al namespace usado en el evento, y es muy útil cuando se han agregado varios listeners al mismo evento, y se necesita hacer algo con sólo uno de ellos.
Retorna Boolean

  • dom: elemento DOM con el evento enlazado.
  • eventname: String que determina el tipo de evento.
  • namespace: String que especifica el namespace del evento.
  setTimeout(function() {
    var txb = $("#txtName").get(0);

    // creamos un event handler sin namespace
    $(txb).click(function() { console.log("> Not namespaced"); });

    // revisa si el event handler con namespace ya fue definido
    var defined = jsu.handlerExist(txb, "click", "fnHighlight");

    // creamos el event handler asignando un namespace al evento
    !defined && $(txb).on("click.fnHighlight", function(e) {
      console.log("Event type:", e.type);
      console.log("Namespace:", e.namespace || e.handleObj.namespace);
      // agregar más código...
    });

    // disparamos el evento específico por su namespace
    $(txb).trigger("click.fnHighlight");
  }, 500);

nsEvents (eventname, namespace)

Construye el nombre del evento, añadiendo un punto y el namespace al final del nombre del evento. Enlazar y desenlazar eventos es un patrón común en la creación de plugins jQuery, de manera que se puedan manejar las acciones ejecutadas por un evento, pero que pasa si tengo más de un event-handler enlazado al mismo evento y quiero hacer algo específicamente con sólo uno de ellos?
Event namespacing es una técnica que provee una forma de administrar event-handlers específicos.
Recomiendo que lean los artículos: Namespace your events y jQuery event names and namespaces
Retorna String

  • eventname: String Lista de eventos separados por espacio.
  • namespace: String Especifica el namespace de los eventos.
  //delega el evento click en el documento
  $(document)
    .off("click.highlight") //quita el event-handler específico por su namespace
    .on(jsu.nsEvents("click", "highlight"), "[data-role=highlight]", function(e) {
        $(this).addClass("jsu-highlight");
    });

  //agrega el mismo event-handler a diferentes eventos
  $(".jsu-maxlength")
    .off(".maxLength") //quita todos los event-handlers por su namespace
    .on(nsEvents("keypress input paste", "maxLength"), function(e) {
        //implementación
    });

fnAddScript (script)

Agrega dinámicamente un script de un archivo (en el mismo dominio). Éste método es útil para inyectar dependencias desde un archivo externo, en caso de que el código pueda fallar si no se ha cargado alguna dependencia. Así por ejemplo, si usted usa una función que dependa del componente kendo.ui.window, puede verificar si dicha dependencia se ha cargado antes de tratar de acceder al componente.
Nota: Por defecto el script es cargado asíncronamente, y el elemento <script> es creado e insertado conforme a la opción before. Si establece async: false o before: null entonces el script es cargado silenciosamente a través de jQuery.ajax y el objeto jqXHR es retornado (provee los métodos diferidos)
Nota: Tenga en cuenta que jQuery.ajax no funciona con el protocolo file:/// (al menos en chrome).
Retorna undefined o jqXHR

  • script: String Object El origen del script a ser cargado.
    Si es un string, contiene la ruta del script a cargar.
    De otro modo, es un objeto con las siguientes propiedades:
    • src: String default: null Especifica la URL del archivo externo.
    • async: Boolean default: true Determina si el script será ejecutado asíncronamente.
    • charset: String default: null Define la codificación de caracteres que utiliza el script.
    • onload: Function default: null Callback que se ejecuta después que el script es cargado.
    • before: String default: "jherax.js" Indica en donde insertar el script. Parte del atributo src de un elemento <script> en donde el nuevo script es insertado. Si establece la propiedad en null el script será cargado silenciosamente.

► module.js (este es el archivo a cargar)

    console.log("Loaded > module.js", date.toISOString());
    return {
        name: "jherax",
        date: date
    }

    }, 500);

    //carga el script síncronamente,
    //es cargado silenciosamente por $.ajax y
    //reacciona cuando el script ha terminado de cargar,
    //mediante el método diferido .done() del objeto jqXHR.

fnEscapeRegExp (text)

Transforma el texto de entrada a un patrón literal en el constructor de una expresión regular.
Esto significa que los caracteres especiales en una expresión regular son tratados como literales.
Por ejemplo, la expresión regular "(A+)" se convertirá en "\(A\+\)"
Retorna String o null si el parámetro text no es un string.

  • text: String que representa una expresión regular.
  var path = "C:\\Documents";
  var re1 = new RegExp(path); //representa una expresión regular
  var re2 = new RegExp(jsu.fnEscapeRegExp(path)); //expresión literal

  console.log("Testing > ", path);
  console.log("re1: " + re1.test(path) + ", RegExp: " + re1.source);
  console.log("re2: " + re2.test(path) + ", RegExp: " + re2.source);

fnIsValidFormat

This Object provides a namespace for the functions validating specific formats.
Each function defined in the namespace receives the value parameter as the text to validate.
Note: Validators are also implemented as a jQuery extension, see jQuery.fnIsValidFormat.
Important The validator functions return Boolean

The value parameter is a String or DOM element inputType:text.

  • time (value) Validates the time format according to timeFormat
  • date (value) Validates the date format according to dateFormat
  • datetime (value) Validates full date format according to dateFormat + timeFormat
  • password (value) Validates the password strength (8+ characters, 1+ uppercase and 1+ number)
  • email (value) Validates an email address.
  • ipv4 (value) Validates an IP address v4.
  • latitude (value) Validates the latitude range from -90 to 90.
  • longitude (value) Validates the longitude range from -180 to 180.
  • set (validator, fn) This function is able to create or redefine a validator function.
    • validator: String with the name of the validator to create or redefine.
    • fn: Function that performs the validation. Should return true or false

    jsu.fnIsValidFormat.set("number", function (text) {
      text = jsu.inputType.isText(text) ? text.value : text.toString();
      var pattern = /^(?:\d+\.)?\d+$/;
      return pattern.test(text);
    });

    //Redefine the validator for email
    jsu.fnIsValidFormat.set("email", function (text) {
      text = jsu.inputType.isText(text) ? text.value : text.toString();
      var pattern = /(\w[-._\w]*\w@\w[-._\w]*\w\.\w{2,3})/i;
      return pattern.test(text);
    });

    //Test the newly created validators
    console.log("number:", jsu.fnIsValidFormat.number("109.35"));
    console.log("email:", jsu.fnIsValidFormat.email("mail@yahoo.com"));
  })();
